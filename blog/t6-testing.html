Technical Blog Part 7


Explain Javascript scope to a non-tech friend.


Scope can kind of be thought of like areas. You have a global scope (everywhere), and you have local scopes (specific to only one area). Lets think of variables as laws; you can have a global variable, which would mean that law applies to everywhere, regardless of a specific states laws. However, laws can change, just like a variable; if someone decides to change the law, it can be changed from anywhere.

In Javascript, this would mean that variable applies to everywhere in the program, if it is declared globally, it is seen everywhere. If we're inside a function and we want to change that variable, we can do that too. However, we can also declare the same name variable INSIDE that function, but it will only be relevant to that function.

For example: If we have a global variable named NUMBER, and set it to the value of 5, the program will see that variable as 5 throughout the entire program. Then if we declare a function that sets the value of NUMBER to 6 and call that function, the value of the global variable NUMBER will become 6. However, if we declare another variable named NUMBER inside the function and give it the value of 6, then that becomes a different variable entirely as it is local to that function.
